## Ранжирование как задача ML

+ есть оценки релевантности на парах $\langle q, d_i \rangle$
	+ возможно, на тройках $\langle q, d_i, ctx_q \rangle$
	+ будем разбивать на обучающую и тестовую выборку
+ это классификация или регрессия?
	+ ни то ни другое (хотя есть способы свести к ним)
+ обычно задача — для произвольной пары документов научиться определять, какой должен быть выше по данному запросу
+ функция потерь — например, количество/доля неправильно отранжированных пар

## Алгоритм Роккио

- Хотим: \
$\vec{q_{opt}} = \mathop{argmax}\limits_{\vec{q}} [sim(\vec{q}, C_r)-sim(\vec{q}, C_{nr})]$
+ $\vec{q_m} = \alpha\vec{q_0} + \beta{1\over{|D_r|}}\sum\limits_{\vec{d_j} \in D_r}\vec{d_j} - \gamma{1\over{|D_{nr}|}}\sum\limits_{\vec{d_j} \in D_{nr}}\vec{d_j}$\
, где:
	+ $q_o$ — оригинальный запрос,
	+ $D_r, D_{nr}$ — множества известных релевантных и нерелевантных документов
	+ $\alpha, \beta, \gamma$ — веса, подбираются

## PageRank

+ вся коллекция — граф
+ качество/важность страницы = вероятности попадания на эту страницу при **случайном блуждании** по ссылкам
+ качество/важность страницы определяется количеством ссылок на нее с других страниц
	+ ссылки с более важной страницы вносят б*о*льший вклад 
+ исходящие ссылки передают важность поровну
	+ то есть если страница $P_i$ имеет важность $R_i$, то каждая из $k$ ссылок с этой страницы несет вес $R_i\over k$
+ с вероятностью $d$ вместо перехода по ссылке — переход на&nbsp;случайную страницу

## Каскадное ранжирование

- проблема: хорошая модель может медленно применяться
+ давайте сделаем несколько стадий
	+ сначала фильтруем более простой моделью (например, булевой)
	+ когда остается не очень много документов, можно применять основную «тяжелую» модель
	+ так можно делать много стадий
+ хорошо вписывается в распределенную архитектуру
